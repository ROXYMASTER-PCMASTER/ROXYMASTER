================================================================================
DIAGNÓSTICO DE PROBLEMAS
================================================================================

PROBLEMA: "No se puede conectar a PCMASTER"
Causas posibles:
  1. PCMASTER no está ejecutándose
  2. Firewall bloqueando puerto 5006
  3. IP o puerto incorrecto en config.json
  4. Red Tailscale no activa

Diagnóstico:
  > netstat -ano | findstr :5006              (¿Puerto abierto?)
  > Get-Process python                         (¿servidor corriendo?)
  > ipconfig | findstr Tailscale              (¿Tailscale activo?)
  > tracert 100.111.179.65                     (¿IP accesible?)

Solución:
  1. Iniciar PCMASTER manualmente
  2. Permitir puerto 5006 en firewall
  3. Verificar IP en config.json
  4. Reactivar Tailscale

---

PROBLEMA: "Timeout esperando respuesta"
Causas posibles:
  1. Conexión lenta o inestable
  2. Timeout muy bajo en config.json
  3. Servidor sobrecargado

Diagnóstico:
  > ping 100.111.179.65                        (¿Latencia?)
  > Get-Content "logs\pcbot.log" | tail       (¿Qué error específico?)

Solución:
  1. Aumentar timeout: "timeout": 15 en config.json
  2. Revisar recursos del servidor
  3. Revisar logs de red

---

PROBLEMA: "Puerto 5006 ya en uso"
Causas posibles:
  1. Servidor anterior no cerró
  2. Otro proceso usando el puerto

Diagnóstico:
  > netstat -ano | findstr :5006              (¿Quién usa el puerto?)

Solución:
  > taskkill /PID <PID> /F                     (Matar proceso)
  O cambiar PUERTO_WS en instalador

---

PROBLEMA: "Error de autenticación en Roxy API"
Causas posibles:
  1. Token incorrecto
  2. Roxy API no disponible
  3. Credenciales expiradas

Diagnóstico:
  > Invoke-WebRequest -Uri "http://127.0.0.1:50000/browser/workspace" -Headers @{"token"="TOKEN"}

Solución:
  1. Verificar token en config.json
  2. Regenerar token desde Roxy
  3. Reiniciar Roxy API
