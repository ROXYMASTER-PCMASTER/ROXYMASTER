DESCRIPCIÓN DEL PROYECTO
================================================================================

ROXYMASTER v6.1 es un sistema distribuido de WebSockets que coordina:
  • PCMASTER (servidor):  Coordina perfiles, genera contexto y distribuye tareas
  • PCBOT (cliente):      Conecta perfiles, ejecuta comandos, inyecta comentarios
  • UI (interfaz web):    Monitorea estado, permite control manual

Arquitectura: Cliente-Servidor con reconexión automática
Protocolo:    WebSocket sobre Tailscale VPN
Lenguaje:     Python 3.9+
Escalabilidad: Teórico 1000+ conexiones simultáneas

PROBLEMAS IDENTIFICADOS Y RESUELTOS
================================================================================

PROBLEMA #1: Condición de carrera en acceso a pcbots
  Criticidad: CRÍTICA
  Solución:  Reemplazar threading.Lock con asyncio.Lock
  Estado:    ✓ RESUELTO

PROBLEMA #2: Falta de validación de entrada
  Criticidad: CRÍTICA
  Solución:  Implementar JSON Schema validation
  Estado:    ✓ RESUELTO

PROBLEMA #3: Reconexión sin límite de intentos
  Criticidad: CRÍTICA
  Solución:  Backoff exponencial 2-32 segundos
  Estado:    ✓ RESUELTO

PROBLEMA #4: Excepciones genéricas (Exception)
  Criticidad: CRÍTICA
  Solución:  Usar tipos específicos (TimeoutError, ConnectionClosed, etc)
  Estado:    ✓ RESUELTO

PROBLEMA #5: Memory leak en JARVIS.memoria_por_url
  Criticidad: CRÍTICA
  Solución:  Implementar TTL en cache
  Estado:    ✓ RESUELTO

PROBLEMA #6: Falta de logging estructurado
  Criticidad: ALTA
  Solución:  Logging con niveles y archivos rotadores
  Estado:    ✓ RESUELTO

PROBLEMA #7: Timeouts no configurados
  Criticidad: ALTA
  Solución:  Timeout explícito en todas las operaciones
  Estado:    ✓ RESUELTO

PROBLEMA #8: Sin sanitización de comentarios
  Criticidad: ALTA
  Solución:  Sanitización con límite de caracteres
  Estado:    ✓ RESUELTO

PROBLEMA #9: Sin validación de URLs
  Criticidad: ALTA
  Solución:  Validar schema y netloc
  Estado:    ✓ RESUELTO

PROBLEMA #10: Rate limiting ausente
  Criticidad: ALTA
  Solución:  Token bucket algorithm implementado
  Estado:    ✓ RESUELTO
